## 1. 배경 (Background)

신혼부부가 **집 장만, 대출 상환, 임신·육아 준비, 큰 목돈 마련**이라는 장기 재무 목표를 함께 관리하려면 수입·지출·저축·대출·투자·일정을 한곳에서 보고 부부가 같이 들여다볼 수 있어야 한다. 기존 가계부 앱은 (1) 부부 공동/개별 구분이 약하고, (2) 가입·과금·광고가 부담스럽고, (3) 일정·메모 등 생활 관리와 분리되어 있다.

이에 **설치·가입 없이 브라우저만으로 동작하고, 부부가 함께 보며, 따뜻한 감성의 단일 웹앱**을 직접 제작한다.

## 2. 목적 (Purpose)

- 부부의 현금 흐름(수입−지출−저축)과 **실제 남은 돈**을 한눈에 파악한다.
- 장기 목표(집·육아·목돈)의 **진행률과 월 적립 필요액**을 자동 계산한다.
- 가계 관련 **일정·메모**를 함께 관리하고 공유한다.
- 두 사람의 기기(PC·모바일)에서 **같은 데이터를 동기화**한다.

## 3. 비즈니스 목표 (Goals)

| # | 목표 | 측정 지표 |
|---|------|-----------|
| G1 | 매월 가계 입력 습관화 | 월 입력 건수 ≥ 부부 합산 30건 |
| G2 | 장기 목표 가시화 | 장기 프로젝트 진행률·월 적립액 자동 산출 |
| G3 | 부부 공동 사용 | 2개 기기 동기화 사용률 100% |
| G4 | 무비용 운영 | 호스팅·DB 모두 무료 티어 내 운영 |

## 4. 범위 (Scope)

### 4.1 In Scope
- 수입/지출/저축/투자/대출/장기 프로젝트/생활비 통장/일정/메모 관리
- 대시보드(요약·편집·정렬), 월별 필터, 원화 표기
- 데이터 백업/복원(JSON), 엑셀 호환(CSV), 초기화
- PIN 잠금, 반응형(모바일/PC), PWA(홈 화면 설치·오프라인)
- 클라우드 동기화(Supabase) 및 설정 링크 공유

### 4.2 Out of Scope
- 은행/카드사 실시간 연동(오픈뱅킹·스크래핑)
- 다중 가구/다중 사용자 계정 체계, 권한 관리
- 실시간 주가·환율 자동 시세 연동
- 세무·신용 평가 등 금융 자문 기능

## 5. 이해관계자 (Stakeholders)

| 구분 | 대상 | 관심사 |
|------|------|--------|
| 주 사용자 | 은정, 성현 (신혼부부) | 쉽고 빠른 입력, 함께 보기, 목표 달성 |
| 제작/운영 | 곽은정 | 무비용·무서버 유지보수, 데이터 안전 |

## 6. 핵심 비즈니스 요구사항 (Business Requirements)

| ID | 요구사항 | 우선순위 |
|----|----------|----------|
| BR-01 | 가입·서버 없이 브라우저만으로 즉시 사용 | 필수 |
| BR-02 | 부부 공동지출/개별지출, 사람별(은정·성현) 구분 | 필수 |
| BR-03 | 실제 보유 현금과 일치하는 "남은 돈"(시작잔액·이월·저축·신용카드 반영) | 필수 |
| BR-04 | 장기 목표 진행률 및 월 적립 필요액 자동 계산 | 필수 |
| BR-05 | PC·모바일 데이터 동기화(부부 공유) | 필수 |
| BR-06 | 데이터 백업/복원 및 분실 대비 | 필수 |
| BR-07 | 가계 일정·메모 통합 관리(달력·반복·공유) | 권장 |
| BR-08 | 오래 써도 질리지 않는 따뜻한 신혼집 감성 UI | 권장 |
| BR-09 | 개인정보·금융정보가 외부에 노출되지 않을 것 | 필수 |

## 7. 가정 및 제약 (Assumptions & Constraints)

- 단일 HTML 파일(외부 프레임워크 미사용), CSS·JS 인라인 포함.
- 데이터는 기기 `localStorage`에 우선 저장 → 사용자 소유·기기 보관.
- 클라우드 동기화는 사용자가 직접 생성한 무료 Supabase 프로젝트 사용(anon key는 공개 가능, 가족 코드가 접근 비밀번호 역할).
- 음력 일정은 브라우저 내장 단기(음력) 달력(Intl) 지원 시 자동 변환.
- 호스팅: GitHub Pages(무료, 공개 저장소). 코드에는 어떠한 비밀정보도 포함하지 않음.

## 8. 리스크 (Risks)

| 리스크 | 영향 | 완화책 |
|--------|------|--------|
| 브라우저 캐시로 업데이트 미반영 | 중 | 서비스워커(network-first) + 설정 내 "최신으로 업데이트" 버튼 |
| 동시 편집 충돌 | 저 | 최종 저장 우선(LWW) + 수동 받기 강제 옵션 |
| 기기 분실/캐시 삭제 시 데이터 손실 | 중 | JSON 백업 권장 + 클라우드 동기화 |
| 가족 코드 유출 | 중 | 긴 무작위 코드 권장, 부부만 공유 |

## 9. 성공 기준 (Success Criteria)

- 부부 두 기기에서 동일 데이터가 동기화되어 보인다.
- 월급(말일)·신용카드(결제일)·저축·대출까지 반영된 "남은 돈"이 실제 통장과 일치한다.
- 장기 목표별 진행률과 월 적립 필요액이 자동으로 표시된다.
- 설치·과금 없이 PC·모바일에서 앱처럼 사용된다.
